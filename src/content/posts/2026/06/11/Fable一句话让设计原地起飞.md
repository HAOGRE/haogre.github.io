> **"把 App 的设计级别比现在提升 100 倍。"**
>
> 就这一句话，它把我的 App 变成了另一个东西。

---

## 那个下午发生了什么

我在做一款 iOS 专注锁屏工具——ZenLock。功能完整，逻辑通畅，StoreKit 接入了，Live Activity 也跑起来了。唯一的问题是：**它看起来太普通了。**

白底、灰卡片、双轮滚动选择器——能用，但没感觉。像一个穿着病号服的产品。

然后我在 Claude Code 里只打了一句话：

> **把 App 的设计级别比现在提升 100 倍。**

注意，我没有给它任何具体的设计方案，没有说用什么颜色、什么组件、什么动效。我只是给了它一个**目标高度**，让它自己想怎么到达。

接下来发生的事让我愣了很久。

---

## 它做了什么

它没有只改几个颜色。它重新构建了整套设计系统：

**从 `ZenTheme.swift` 开始**，新增了极光色板（Indigo / Violet / Blue / Teal 四个自适应色调）、玻璃态 Token（`zenGlassFill` / `zenGlassStroke`）、间距与圆角枚举、两个动效常量（`zenBreath` 4秒循环呼吸、`zenSoft` 弹簧动效）。

然后它创造了五个全新组件，放进 `Shared/Components/`：

| 组件 | 作用 |
|------|------|
| `AuroraBackground` | 多色光斑在屏幕上缓慢漂移，有 calm/deep 两种氛围，`reduceMotion` 时自动回退为静态渐变 |
| `GlassCard` | 统一的玻璃磨砂背景修饰器，带描边与阴影 |
| `BreathingRing` | 计时器外圈渐变进度环 + 呼吸节奏光晕 |
| `RadialDurationDial` | 可拖拽圆形时长选择器，替代双滚轮，带刻度与触觉反馈 |
| `PrimaryButton` | 统一渐变主操作按钮，带按压缩放与触觉 |

然后它把所有页面重新铺了一遍——空闲页、专注中、确认 Sheet、付费墙、Live Activity——**所有逻辑文件一行没动，只是视觉层换了皮肤**。

整个过程，我喝了一杯咖啡。

## 为什么我觉得这件事值得记录

不是因为设计变漂亮了——虽然它确实漂亮了很多。

而是因为这件事揭示了一种**新的开发节奏**：

过去，UI 升级意味着：  
→ 设计师出稿 → 评审 → 切图 → 开发实现 → 反复对齐

现在，它意味着：  
→ 你描述一个方向 → Fable 给你一套完整的设计系统实现 → 你 Review、微调

**Fable 的理解能力让人震惊的不只是"能写代码"，而是它能把"100倍"这个模糊的目标，翻译成一套具体的设计决策。**

它没有问我"100倍是什么意思"，而是自己判断：极光渐变背景意味着多层模糊光斑的漂移动画；玻璃态意味着 `.ultraThinMaterial` + 渐变描边高光；呼吸感意味着与用户放松状态呼应的 4 秒循环动效；还有 `reduceMotion` 的无障碍降级——这些细节，它都想到了，而我只说了"100倍"。

这才是重点：**你给一个方向，它完成整条路。**

这不是"AI 辅助开发"，这是**AI 和开发者共同完成了一次真正意义上的产品升级**。

---

## 一点感慨

我做了很多年独立开发，深知"功能可以一个人搞定，但设计往往是短板"。不是不会审美，而是实现成本太高——尤其是动效系统、设计 Token、组件封装这些有门槛的工程工作。

今天这一下，让我重新感受到了当初为什么喜欢做产品：**你脑子里有个画面，它帮你把那个画面变成真的**。

一句话，一杯咖啡的时间，应用就变成了另一个样子。

这就是 Fable 今天给我的体感。
